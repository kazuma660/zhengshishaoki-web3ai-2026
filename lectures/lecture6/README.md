# 第6回 プロトタイプ v2 — TaskLog（「気になってる事を置く場所」へ刷新）

v1（lecture5）の「タスク作っても結局やらない」問題を、**やる重みを極限まで軽くする** 方向で作り直した。

## v1 で感じた違和感
- 「タスク追加 → 優先度設定 → 期限設定」が**宣言の儀式**になっていて重い
- 完了/未完了の二項対立で、中途半端な進捗を扱えない
- 放置タスクが罪悪感の塊として溜まる
- 「1分でできる」フレームは借り物っぽく、本質（やる重み）を外している

## v2 のコンセプト
> 気になってる事を置く場所。触れたら積み上がるし、手放しても咎められない。

「タスク」ではなく「気になってる事」。完了させる対象ではなく、頭の中にあるものの棚。

## v1 からの主な差分
| | v1 | v2 |
|---|---|---|
| 入力 | タイトル + 重要度 + 期限を必須UIで提示 | タイトルのみ。重要度・期限は詳細展開で**後付け**（隠し機能） |
| 完了モデル | チェック → タスク消滅 | **「触った」を回数で積む**（1日無制限・グラデーション） |
| 諦め | 削除（罪悪感） | **「手放す」を一級操作**化、別セクションに残し戻せる |
| 放置 | 見えない | `n日触ってない` バッジで可視化 |
| ログ | 過去30日ヒートマップ | **過去12週ヒートマップ（GitHub風 7×12）** |
| データ | `tasklog:tasks` / `tasklog:done` | `tasklog:items:v2`（v1 データと共存） |

## 動くか
- 「置く」フォーム（タイトルのみ）
- 「触った」ボタン（無制限・ドット表示で積み上げ）
- 詳細パネルで重要度/期限/手放す
- 12週ヒートマップ + 連続日数 / 今日 / 累計
- 手放したものセクション（戻す・消すあり）
- localStorage で永続化

## スタック
Next.js 16 + TypeScript + Tailwind CSS + localStorage
