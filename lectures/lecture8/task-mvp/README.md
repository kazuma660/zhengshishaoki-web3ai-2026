# TaskLog v4 — 最初のワンステップで、何もしない日を作らない

前夜に明日やる事を置く。最初の一歩が目の前にある状態で朝を迎えよう。

- **置く / 明日やる** — 2ボタン。軽い思いつきは「明日やる」で摩擦ゼロで今日に入る
- **細かくする** — 抽象タスクを子タスクに分解（任意の深さ）。「TOEIC → 英単語20個」のように **数えられる粒度** に落とす
- **▼ / ▶** — 子が増えたら畳んで視界から消せる
- **今日やる事** — 朝、判断ゼロで動ける。「最初の一歩」を入力すると着火点が見える
- **やった** — 押すと記録、達成感

## なぜ v3 ではなく v4 なのか

v3 までは「タスクを管理する」発想。けど自分の本当の問題は **「何もしない日を作りたくない」** だった。

```
学校・進路（重い・抽象）を考える → ループ（正解わからない／全部やらなきゃ／時間ない）
  → 疲れる → 軽いタスクにも手が出ない → 何もしない日 → 数ヶ月積もって目標が遠ざかる
```

これを断つには「タスク管理」より **「ループの外に出るドア」** が要る。v4 はそれを目的にした。

## v3 → v4 の主な差分

| | v3 | v4 |
|---|---|---|
| 階層 | フラット | **親→子の任意の深さ** |
| 抽象→具体 | 対処なし | **「細かくする」で数えられる粒度に落とす** |
| 動き出し | 「1分だけ」ボタン押下 | **「明日やる」で前夜セット、朝は目の前にある** |
| AI | ルールベース or API | **削除**（外部AIで分解は十分） |
| 重要度・期限 | あり | **削除**（ループ誘発するため） |
| ヒートマップ | 表示あり | 裏で記録のみ |
