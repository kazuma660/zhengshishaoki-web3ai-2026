## 自分のバグ（v4 が解こうとしている問題）

**何もしない日を作りたくない。** やりたいことも、1年後を見据えてやらなきゃいけないことも頭にあるのに、動けない日がある。

その正体を分解するとループになっていた：

```
学校の課題・進路系（重い・抽象）を考える
  → 「正解わからない／全部やらなきゃ／時間ない」が回る
  → 疲れる
  → 軽いタスク（英語・基本情報）にも手が出なくなる
  → 何もしない日
  → 数ヶ月積もって目標が遠ざかる
  → 焦って戻る …
```

**抽象タスクは動けない**。「TOEICの勉強」だと終わりが見えない。「英単語20個」だと終わる。
**数えられる粒度**に落とせれば動けるし、**最初の一歩**が目の前にあれば動き出せる（特に朝、これが1日の活動量を決める感覚がある）。

## v3 → v4 の主な差分

| | v3 | v4 |
|---|---|---|
| 抽象タスクの扱い | 1階層のリスト、対処なし | **親→子の階層構造**。「細かくする」で**任意の深さに分解**できる |
| 動き出しのレバー | 「1分だけ」ボタン（AI/ルールベース） | **前夜セット → 朝に「今日やる事＋最初の一歩」が目の前** |
| 一発操作 | 「置く」のみ | **「置く」/「明日やる」の2ボタン**。軽い思いつきは摩擦ゼロで今日に入る |
| AI連携 | ルールベースの「1分だけ」あり | **削除**（外部AIで分解は十分、自前API入れる価値が無いと判断） |
| 重要度・期限 | あり | **削除**（「全部やらなきゃ＋優先付けで判断停止」ループを誘発する側だと判断） |
| ヒートマップ | 表示あり | 裏でtouches記録のみ。表示UIは削除（リワード弱かった反省） |

## 設計思想（目的との整合）

v4 は「タスク管理ツール」ではなく**「何もしない日を作らない装置」**。

- **プールに全部出す**：頭の中で考え続けなくて済む
- **「細かくする」で抽象を数えられる単位に**：ループ脱出の主装置
- **▼/▶ で畳める**：重いやつを視界から消せる（「見ないといけないけど軽くしたい」）
- **明日やるはリーフ（子を持たない単位）にだけ**：親は眺める専、動くのは子
- **朝、最初の一歩が目の前**：判断ゼロで開始

## 動くか

- 置く / 明日やる 一発 / 細かくする（任意の深さ）/ 折り畳み / 削除（カスケード）/ 最初の一歩 / やった、全フロー動作確認済み
- 親子は localStorage に `parentId` で表現、データは v3 と後方互換

## スタック

Next.js 16 + TypeScript + Tailwind CSS + localStorage（開発は Claude Code と共同）

## 価値検証（このlectureの本題、これから）

提出が遅れたため、価値検証は**提出後に追記**する形：

- [ ] 自分で 6/18〜7/16 使う → 「何もしない日」が減ったか、ヒートマップで定量
- [ ] 友達 2〜3人に見せる → 「同じ痛みあるか」「これで効きそうか」を中心にヒアリング
- [ ] 検証結果をDiscord投稿 → リンクをここに追記

検証で出た声を v4.1 の判断材料にする予定：
- 「重要度欲しい」声が出るか（出たら復活検討）
- 「今週やる事」枠の要望
- 「最初の一歩を考えるのが重い」→ 細分化AI再導入の検討
