# 第7回 プロダクト v3 — TaskLog（動けない日の「1分だけ」）

🚀 **動くプロダクト: https://task-mvp-chi.vercel.app**

提出パターン: **A-1 プロダクト系**（Vercelデプロイ + GitHub）

## v2 を作って・使ってみての発見

**作った本人が、アプリを開かなかった。**

「タスク作っても結局やらない」バグを直すために作ったアプリで、同じバグが再現した。
原因を分解すると3つ：

1. **開くきっかけがない** — アプリの存在を思い出す動線が日常にない
2. **スマホで使いにくい** — 気になる事が浮かぶのはスマホを触ってる時なのに、PCでしか開かない
3. **リワードが弱い** — 草が生えても嬉しさが薄く、開く動機にならない

## v3 のコンセプト

> 置くまではv2で軽くなった。v3は「置いた後、動けない日でも一歩出せる」に踏み込む。

## v2 からの主な差分

| | v2 | v3 |
|---|---|---|
| 動けない日 | 何も助けてくれない | **「1分だけ」ボタン**。AI（claude-haiku-4-5）がそのタスクの「最初の1分で物理的に完了する行動」を3つ提案 → 選ぶと🪜が付く → 「やった」で草が生える |
| AI | なし | Anthropic Messages API + structured outputs。APIキー未設定でも内蔵ロジックで動くフォールバック付き |
| スマホ | レスポンシブのみ | **PWA対応**（manifest + アイコン）でホーム画面から1タップ。iOSの入力ズーム問題も修正 |
| リワード | 草のみ | 1分の前進にトースト（「1分の前進 🌱」）で即時反応 |

VPCで★を付けていた Pain Reliever「**1分ミニタスク機能**」を、AIの力でようやく回収した形。

## 動くか

- 「1分だけ」→ AI提案3つ → 選択 → 🪜表示 → 「やった」→ 草+トースト（全フロー動作確認済み）
- `/api/breakdown` Route Handler（POST、JSON: `{title}` → `{steps[], source}`）
- v2の全機能（置く・触った・手放す・ヒートマップ・詳細）はそのまま

## スタック

Next.js 16 + TypeScript + Tailwind CSS + localStorage + Anthropic SDK（@anthropic-ai/sdk）

## ローカル起動

```bash
cd task-mvp
npm install
npm run dev
```

AI提案を本物にする場合は `ANTHROPIC_API_KEY` を環境変数に設定（無くても内蔵フォールバックで動く）。
