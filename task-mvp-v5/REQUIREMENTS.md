# TaskLog 要件定義（v1・2026-06-24 再定義）

> 目的レベルから引き直した版。迷ったら [PURPOSE.md](PURPOSE.md) → この順で読む。
> 実装はこの要件に従う。新機能は必ず PURPOSE.md「機能追加の判断軸」に照らす。

## 0. 目的（動かさない軸）

頭で処理しすぎて止まる自分のための「**タスクを外部化する装置**」。

```
重い/抽象を考えすぎ → 「全部やらなきゃ/正解わからない/時間ない」ループ → 疲れる
→ 軽いタスクも手が出ない → 何もしない日
```

これを断つ2本柱：
1. **外部化**：頭の中の負荷を外に出す
2. **摩擦低減**：動ける状態への距離を縮める（抽象→数えられる粒度／最初の一歩）

やる気管理・疲労・メンタルは**範囲外**（別アプリ）。

## 1. 利用シーン（要件の出発点）

| いつ | やること | 効かせる柱 |
|---|---|---|
| 随時 | 浮かんだ気になる事を放り込む | ①外部化 |
| 夜 | 明日やる事を数件セット | ②摩擦低減（朝の判断ゼロ化） |
| 朝 | 今日やる事を見て「最初の一歩」を書いて動く | ②摩擦低減 |
| 日中 | できたら「やった」／無理なら明日へ or 放置 | ①② |
| 日付変わり | 明日→今日 自動繰越、未完了の今日→未達成 | 仕組み化 |
| 弱い日 | 何もしなくてOK。未達成は責めない記録 | やらない原則 |

## 2. セクションと役割

| セクション | 役割 | 階層 |
|---|---|---|
| **気になってる事** | 抽象・長期のバックログ（留学/進学/授業など、1〜2日で終わらないもの）。**「細かく」で数えられる粒度に分解する“考える場所”** | あり |
| **今日やる事** | 取り出した具体的な実行アイテム。「最初の一歩」を書く“動く場所” | なし |
| **明日やる事** | 前夜セット。日付変わりで自動的に今日へ | なし |
| **未達成** | 今日の未完了が落ちる先。**既定で畳み**・責めない記録 | なし |
| **やったこと** | 完了アーカイブ（7日でアーカイブ、365日で自動削除） | なし |

中心フロー：`気になる(留学) → 細かく → 子(IELTS申込) → 今日/明日に出す → 今日↔明日でずらす`

## 3. 操作モデル

### 追加（外部化）
- 上の入力＋「置く / 今日やる / 明日やる」。Enter＝置く。
- **IME変換確定のEnterで誤submitしない**（`isComposing` ガード）。

### 分解（摩擦低減）
- 気になる項目の「＋」で数えられる子に割る。子を今日/明日へ出せる。
- **「＋（細かく）」の入力欄は同時に最大1つ**。

### 画面（タブ）と移動・階層化 — ドラッグは廃止、タップと折りたたみだけ
- **タブUI**：`気になる / 今日 / 明日` を1モードずつ表示（スマホ=下タブ・親指圏／PC=上タブ・中央寄せ、起動時=今日）。未達成・やったことは今日タブ末尾に畳んで同居。考える場所(気になる)と動く場所(今日)を分離。
- **ドラッグ＆ドロップは入れない**（過去に試したが、移動が「ハマらない／長押しが煩わしい」＝摩擦になったため撤去）。操作はタップと折りたたみのみ：
  - 階層化（子タスク）：気になる項目の **＋（細かく）** で作る／**▼/▶** で畳む。
  - セクション間移動：**今日 / 明日 / 今日へ / 明日へ / 外す** ボタン（タブをまたいでもボタンで完結）。
- **並び替え（リスト内で順番を決める）は入れない**（優先付け→「全部やらなきゃ」ループ誘発のため）。
- 入力欄では IME 変換確定の Enter で誤submitしない。

### 完了・繰越
- 「やった」→ やったこと。
- 日付変わり：明日→今日 繰越、未完了の今日→未達成。未達成は既定で畳み。

### 同期・ログイン（インフラ）
- Firestore `/users/{uid}` に `{items, lastSeen}`、`onSnapshot` でリアルタイム同期。
- ログイン：**Google**＋**メール/パスワード**（メール方式はFirebaseコンソールでEmail/Password有効化が前提）。

## 4. やらない（スコープ外）

- リスト内の手動並び替え・重要度/優先付け（→「全部やらなきゃ」ループを誘発）
- 連続記録の煽り（◯日連続！）
- やる気管理／疲労トラッキング／メンタル管理／AIで「最低限」判定
- 「やる気0でも動かす」（0の日は休んでOK）

## 5. 表示まわり

- PCの広い幅（xl）では **各列が独立スクロール**（気になるが長くても今日/明日が画面外に押し出されない）。スマホは縦積み＋ページ全体スクロール。
- 折り畳み（▼/▶）で重いものを視界から消せる。
