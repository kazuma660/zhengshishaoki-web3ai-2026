# TaskLog の目的（迷ったらここに戻る）

## 一言で

**頭の中で処理しすぎて止まる自分のための、タスクを外部化する装置。**

## 解こうとしてる自分のバグ

```
学校・進路系（重い・抽象）を考えすぎる
  → 「正解わからない／全部やらなきゃ／時間ない」のループ
  → 疲れる
  → 軽いタスク（英語・基本情報）にも手が出ない
  → 何もしない日
  → 数ヶ月積もって目標が遠ざかる
```

このループから抜けるための「外に出るドア」を作る。

## 設計の2つの核

1. **タスクを外部化する**：頭の中で処理し続けなくて済むように、外に出す
2. **動ける状態にする時の摩擦を下げる**：抽象→数えられる粒度に落とす／最初の一歩を見せる

これ以外（やる気管理／疲労回復／メンタル管理）は **別アプリの仕事**。混ぜると壊れる。

## 主要機能 × 目的の対応

| 機能 | 目的のどこに効くか |
|---|---|
| 置く / 今日やる / 明日やる の3ボタン | 軽い思いつきを摩擦ゼロで頭から出す |
| 親→子の細分化（任意の深さ） | 抽象タスクを「数えられる粒度」に分解 |
| ▼/▶ 折り畳み | 重いやつを視界から消す |
| 今日やる事 + 最初の一歩 | 朝、判断ゼロで動ける |
| 明日やる事 + 日付繰越 | 前夜セット、朝の判断ゼロ化を仕組み化 |
| やったこと（アーカイブ込み） | 達成感、何もしない日の対極 |
| 未達成 section | やれなかった日を淡々と記録、責めない |
| 編集機能 | 外部化したものを後から精緻化できる |
| DnD（ドラッグ&ドロップ） | 階層移動を直感的に |

## やらない／入れない原則

- **やる気の管理**：精神論ツールにしない
- **疲労トラッキング**：体力管理アプリの仕事
- **AIで「最低限を判定」する系**：頭使わせない目的で頭を使わせるツール（AIなしで運用）
- **連続記録の煽り**（◯日連続！）：プレッシャー化する
- **ループを誘発する機能**：重要度・優先付け（→「全部やらなきゃ」を強化する）
- **「やる気0でも動かす」**：これは目的の外側。0の日は休んでOK

## 機能追加するときの判断軸

新しい機能を入れたくなったら、これに照らす：

1. **外部化に効くか？**（頭の中の負荷を下げるか）
2. **摩擦低減に効くか？**（動ける状態への距離を縮めるか）
3. **「全部やらなきゃ／正解わからない／時間ない」のループを誘発しないか？**
4. **依存を増やしてないか？**（軽量さを保ててるか）

3つ目NOなら却下。1〜2のどちらかが YES なら検討。

## 検証の問い

使ってる時に時々これを自分に問う：

- **何もしない日が減ったか？**
- **「考えすぎて止まる」時間が減ったか？**
- **使うこと自体が新しい摩擦になってないか？**（本末転倒チェック）

## 拡張の方向性（v5以降の候補、今はやらない）

- **振り返り装置**：v4の蓄積データで自分のパターンを観察するアプリ
- **MOE指標**：「何もしない日カウント」「動いた日カウント」の自動表示

これらは v4 と別アプリとして作る方向。v4 に混ぜない。
