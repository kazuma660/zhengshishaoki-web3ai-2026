## v5.2.0 — 2026-06-25

UI再設計（タブ化）＋ドラッグ撤去。「ごちゃつき」解消と「すぐ動ける」最短化。
要件は [REQUIREMENTS.md](REQUIREMENTS.md)、デプロイ手順は [DEPLOY.md](DEPLOY.md)。

### 変更
- **タブUIに再設計**：4列同時表示を廃止し、`気になる / 今日 / 明日` の3タブで1モードずつ表示（スマホ=下タブ・親指圏／PC=上タブ・中央寄せ、レスポンシブ）。起動時は **今日タブ**。考える場所(気になる)と動く場所(今日)を分離し、実行中に長期の重い項目が視界に入らない＝「全部やらなきゃ」ループを抑制。
- **未達成・やったこと**を今日タブ末尾に畳んで同居（タブを増やさない）。
- **「置く」ボタン→「気になる」に改名**（送り先タブ名と一致させ、入力→行き先の対応を明確化）。
- タイトル **「TaskLog v5」→「TaskLog」**（内部バージョンは継続）。

### 削除
- **ドラッグ＆ドロップを完全撤去**（`@dnd-kit/core` 不使用に）。操作はタップ＋折りたたみのみ。
  - 気になるの階層化は **＋（細かく）** で作る、**▼/▶** で畳む。セクション間移動は **今日/明日/外す** ボタン。
  - 経緯：移動D&Dは「ハマらない／長押しが煩わしい」、設計の核（すぐ動ける・摩擦低減）に対してドラッグが摩擦になっていたため、ボタン＋折りたたみへ一本化。

### 修正
- **やったことの「戻す」**：完了した**元の場所（今日/明日）に戻る**よう修正（従来は気になるに落ちていた）。完了時に `isToday/isTomorrow` を保持。
- 完了済みの「明日」項目が日付変わりで「今日」に化けないようガード（完了済みはやったことに留まる）。

### 内部
- `@dnd-kit/core` は依存に残るが未使用（将来削除可）。
- タブ状態は localStorage `tasklog:tab:v1` に保存。

## v5.1.0 — 2026-06-24

ヒアリング8項目の反映＋ドラッグ操作の再設計（本番反映済み）。
要件は [REQUIREMENTS.md](REQUIREMENTS.md)、デプロイ手順は [DEPLOY.md](DEPLOY.md) を参照。

### 追加
- **メール/パスワードログイン**：Google に加えて追加（要：Firebase で Email/Password プロバイダ有効化）。ログイン⇔新規登録トグル、日本語エラー表示
- **セクション間ドラッグ移動**：カードを掴んで 今日↔明日↔気になる を移動。**落とす場所で決まる**（別項目に重ねる＝子タスク化／列の余白＝移動／最上位ゾーン＝親外し）。移動ボタンは併存で維持
- **各列の独立スクロール**（PC・xl幅）：気になるが長くても今日/明日が画面外に押し出されない
- **ログアウト確認ポップアップ**：誤タップでの再認証を防止
- **未達成の気づき通知**：日付が変わった初回だけ、繰越トーストに「未達成 N件」を添える

### 変更
- **未達成セクションをデフォルト畳み**：起動時の圧を下げる。常時表示は廃止（畳んだ見出しに件数は出る）。UI設定キー `tasklog:ui:v1`→`v2`
- **「細かく」を ＋/－ アイコン化**：横幅節約、スマホでタイトルが読みやすく
- **「細かく」入力欄は同時に最大1つ**（書きかけは保持）
- 今日/明日カードを**カード全体ドラッグ可能**に（⋮⋮ ハンドル限定をやめた）

### 修正
- **IME 変換確定の Enter 誤submit**：`isComposing`/`keyCode 229` ガードを3入力欄（置く・子追加・編集）に追加
- ドラッグ自己レビューで検出した不具合：タッチでボタン/入力欄から誤ドラッグ（`onTouchStart` も stopPropagation）、pool へ戻す時の `parentId`/`miniStep` 残留（独立カード化）、ほか

### 内部
- `@dnd-kit/core` の `collisionDetection` を pointer ベースのカスタム `dropCollision` に（項目を列ゾーンより優先＝「重ねたら子／余白なら移動」を確定）
- 4レンズの自己レビューワークフローでドラッグ改修を検証
- `REQUIREMENTS.md`（要件再定義 v1）・`DEPLOY.md`（デプロイ手順）を追加

## v5.0.2 — 2026-06-23

ユーザーヒアリング後のフィードバック反映（Phase 1 + 横並び）

### 追加
- **Enter キーで追加**：メイン入力欄＆子追加入力欄、Enter で「置く」「子追加」できる
- **階層 5層制限**：これ以上深くできない。5層目では「細かく」ボタン非表示、DnD でも超えそうな時はトースト
- **子タスクに親コンテキスト表示**：今日/明日/やったこと/未達成セクションで、子は「眠い / 目を閉じる」のように親チェーンが上に小さく出る
- **横並びレイアウト（デスクトップ）**：1280px 以上で4カラム（今日 ｜ 未達成 ｜ 気になる ｜ 明日）
- **セクション折り畳み**：各セクションに ▼/▶ ボタン、状態は localStorage で記憶
- **未達成セクション常時表示**：空でも「やれなかった日は淡々と記録、責めない」プレースホルダ

### 変更
- インデント線・×印・⋮⋮ハンドルの色を明るく（暗背景での視認性 ↑）
- 親コンテキスト文字サイズ：10px → 12px、色を少し明るく
- ヘッダーサブタイトル「前夜に明日やる事を置く...」削除
- フッター「TaskLog v5 — 最初のワンステップで...」削除

### 修正
- PWA インストール時、上下スクロールで白い下地が見える問題（html/body に背景色追加）

### 内部
- `showToast` を `useCallback` でトップに移動（hoisting 依存削除）
- セクション折り畳み状態を localStorage キー `tasklog:ui:v1` に保存

### 保留中（次回の判断軸：PURPOSE.md §「機能追加するときの判断軸」）
- **E：プリセット保存機能**（毎日呼び出す用）— ルーチン強制化の懸念。「責めない設計」に絞れば採用可
- **I：子ができた親から「今日やる」削除**— 抽象タスクで止まるバグ防止には効くが、設計変更として大きい
- **H：エクスポート / インポート機能**— クラウドバックアップで一旦充足、v5.1 以降

## v5.0.0 — 2026-06-22

Firebase 化・マルチユーザー対応（メジャーリリース）

### 追加
- **Google ログイン**（Firebase Authentication）
- **Firestore でデータ保存**：`/users/{uid}` に `{items: Item[], lastSeen: string}`
- **複数端末同期**：`onSnapshot` でリアルタイム反映
- **マイグレーション処理**：初回ログイン時、同一ドメインの localStorage 旧データがあれば一度だけアップ

### 変更
- データ保管：localStorage → Firestore
- バージョン表記：v4.3 → v5

### 内部
- 書き込みは 500ms debounce、JSON 比較でループ防止
- セキュリティルール：本人のみ自分のドキュメントを読み書き可能
- 新規 Vercel プロジェクト（task-mvp-v5.vercel.app）
- v5-dev ブランチで GitHub に push

### 動機（記録として）
- 配布想定
- データ改竄予防
- 複数端末同期（PC ↔ スマホ）
- データのクラウドバックアップ
- localStorage の容量圧迫阻止
